What is Git?

Git is a version control system used to track changes in files, primarily source code, during software development.

  • Tracks changes in source code over time
  • Enables collaboration among developers
  • Maintains a detailed history of changes
It was created by Linus Torvalds in 2005 to support the development of the Linux kernel.

git --version

Git allows multiple developers to collaborate efficiently, providing tools to manage code versions, merge changes, and resolve conflicts. It helps keep a detailed history of every modification made to a project.

git help

One of Git’s main advantages is that it is a distributed system, meaning each user has a full copy of the entire repository, including its complete history and branches.

git config --global user.name "Your Name"

Installing Git

Installing Git is straightforward and available for all major operating systems. Users can download the latest version from the official Git website or use a package manager.

sudo apt install git

On Windows, Git can be installed via the Git for Windows installer, which includes Git Bash, a terminal emulator for Git commands.

  • Available for Windows, macOS, and Linux<
  • Install via package managers like apt, brew, or yum
  • Configure user name and email after installation
On macOS, Homebrew is a common choice for installation.

brew install git

Linux users can install Git using the default package manager of their distribution. It’s recommended to always install the latest stable version for access to the newest features and fixes.

git config --global user.email "you@example.com"

Basic Git Commands

Git provides a variety of commands for version control operations.

  • git init: Initialize a new repository
  • git add: Stage changes for commit
  • git log: View commit history
Some of the most commonly used include git init, git add, git commit, and git status.

git init

These commands help initialize a repository, stage changes, save snapshots of your codebase, and check the current state of your working directory and staging area.

git add .

By mastering these basic commands, developers can manage their workflows more efficiently, ensuring that each change is properly tracked and documented.

git commit -m "Initial commit"

Working with branches

Branches in Git allow developers to work on different features or bug fixes independently. Each branch is a separate line of development.

git branch new-feature

Using branches makes it easier to experiment with new ideas without affecting the main codebase.

  • Create new branches to develop features
  • Merge branches to integrate changes
  • Use branching workflows like Git Flow
Once a feature is complete, it can be merged back into the main branch.

git checkout new-feature

This branching model supports collaboration, parallel development, and better code management — especially in teams following workflows like Git Flow or GitHub Flow.

git merge new-feature

Collaborating via Pull Requests

GitHub is a web-based platform built around Git that provides remote repositories, issue tracking, and collaboration features like pull requests and code reviews.

git remote add origin https://github.com/yourusername/repo.git

It allows developers to share their code, collaborate on projects, and contribute to open-source software.

  • Fork repositories to create your own copy
  • Push changes to your remote branch
  • Create pull requests to propose changes
GitHub also integrates with CI/CD tools and project management systems.

git push -u origin main

Using GitHub with Git gives teams a powerful toolkit for managing software development, from individual experiments to large-scale applications with many contributors.

git clone https://github.com/otheruser/project.git
